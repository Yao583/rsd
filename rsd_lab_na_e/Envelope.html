{% trans "Implementation" %}

{% blocktrans trimmed %} Both treatments begin like this: {% endblocktrans %}

You will submit your ranking, by placing the prizes in numbered "envelopes":

  • In Envelope 1, you will place the prize you want to rank first.
  • In Envelope 2, you will place the prize you want to rank second, and so on.
  • All of the envelopes must have different prizes. You can only rank a prize once.
  • You can submit up to four envelopes, same as the number of prizes.
  • If you choose to submit less than four envelopes, you must fill the lower-numbered envelopes first.

{% trans "How the Interface Works" %}

The interface does not know how many envelopes you submitted nor does it know what is in each envelope.

The interface is the same for everyone.

When it is your turn, the interface knows which prizes are available, but keeps no record of any envelopes previously considered. It will consider your envelopes before turning to the next participant in line.

The interface is designed to only be able to consider one of your envelopes at a time. When it considers an envelope the interface:

  1. Does not remember previous envelopes from the same person.
  2. Does not know how many envelopes are still to come or what is in them.

Since the interface can only consider one envelope at a time, we will refer to the envelope that the interface observes as the envelope under consideration.

{% trans "The Allocation Process" %}

When it is your turn, the interface checks whether you submitted Envelope 1:

  • If you did not submit Envelope 1, you do not receive any prize and the interface turns to the next person.
  • If you did submit Envelope 1, then Envelope 1 becomes the envelope under consideration. The interface will open the envelope under consideration:
    • If the prize in the envelope under consideration is available, you will win this prize. The interface no longer can access any of your envelopes that it did not need to open.
    • If the prize in the envelope under consideration is unavailable, then the following takes place:
      • The interface will destroy Envelope 1 and forget the prize you indicated in that envelope.
      • The interface then checks if you submitted Envelope 2.

If you did not submit Envelope 2, you do not receive any prize and the interface turns to the next person.

If you did submit Envelope 2, then Envelope 2 becomes the envelope under consideration:

  • If the prize in the envelope under consideration is available, you will win this prize. The interface no longer can access any of your envelopes that it did not need to open.
  • If the prize in the envelope under consideration is unavailable, then the following takes place:
    • The interface will destroy Envelope 2 and forget the prize you indicated in that envelope.
    • The interface then checks if you submitted Envelope 3 and so on.

{% trans "End of Process" %}

At the end of this process the interface was successful in giving you one of the prizes from the envelope under consideration or unsuccessful. That is, the interface ran out of your envelopes and could not assign you a prize.

In either case, the interface moves on to the next person. Once the interface went through all the people the process stops.

{% trans "General" %}

At the end of the allocation process, each participant who is not allocated a prize is randomly allocated one of the prizes that have not been allocated yet.
